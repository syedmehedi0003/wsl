01 Windows Setup – WSL (Windows Subsystem for Linux) ▼
WSL = Windows Subsystem for Linux — run Linux commands inside Windows.
First enable: Turn Windows features on/off → check WSL
WSL Commands
➜wsl --helpshow help
➜wsl --installinstall WSL
➜wsl --listlist installed distros
➜wsl --list --onlinelist available distros online
➜wsl.exe --install Ubuntuinstall Ubuntu (old way)
➜wsl --install -d Ubuntuinstall Ubuntu (preferred)
➜wsl --listverify installation
➜wsl --install -d Ubuntu-20.04install Ubuntu 20.04 specifically
➜wsl --listfinal check
Also: Microsoft Store → Ubuntu → Install as an alternative method.

03 Git Basics – The 3 Stages ▼
Git is a version control system.
3 Stages of Git (after git init)
①
Working Area
Unstaged — RED
→
git add .
②
Staging Area
Staged — GREEN
→
git commit -m
③
Version Created
Commit saved
Initialise a Repository
➜cd ..go to parent directory
➜mkdir git-playgroundcreate project folder
➜cd git-playgroundenter the folder
➜git initinitialise Git repository
➜ls -asee hidden .git folder
➜touch index.htmlcreate a file
➜lsverify file exists
➜git statuscheck current Git state
➜git add index.htmlstage specific file
➜git statusverify file is staged (green)

06 Versions, Checkout & Reset ▼
Creating Multiple Versions
➜git commit -m "version 1"create version 1 snapshot ★★
➜git logcheck version (shows commit hash/id)
➜touch hello2.txtcreate new file
➜ls
➜git status
➜git add hello2.txtstage specific file (or multiple files)
➜git add .OR stage all — পাশা পাশি multiple file add করা যায়
➜git commit -m "version 2"create version 2
➜git log --onelinesee both versions with commit IDs ⇒ commit hash/id ↑
Navigating Between Versions
➜git checkout 81966bajump to specific commit (commit id দিয়ে নির্দিষ্ট commit-এ যাওয়া যায়)
➜git logonly show hero section commit
➜git status
➜git log --onelineshow all from this point
➜git checkout masterreturn to latest (or: git checkout commit_id)
➜git logall commits visible again
Resetting Commits ⚠️
⚠ WARNING: git reset --hard permanently removes commits. এটি দিয়ে permanently কোন একটি commit-এ back করা যায়।
➜git reset --hard 81966bapermanently go back to this commit (removes later commits)
➜git logverify – later commits gone, shows only up to "footer"
➜git reflog★★ safety net – shows ALL actions including deleted commits
➜git reset --hard <commit-id>recover using hash from reflog
git log → git reflog ★★ — use reflog to recover from accidental resets!
07 Branching & Merge Conflicts ▼
Branch Switching
➜git log --onelineview commit history
➜git checkout masterswitch to master
➜git checkout ITBranchswitch to ITBranch
➜git checkout mainswitch to main branch [main branch এ থাকতে হবে]
➜git statuscheck current branch & state
➜lslist files
➜cat conflict.txtview file contents
➜vim conflict.txtedit file (i, Esc, :wq)
➜git add .
➜git status
Merge Conflict Lab – Ubuntu Simulation
Simulate two developers changing the same line. Run on Ubuntu / AWS EC2.
➜sudo apt updatewelcome to ubuntu
➜sudo apt install gitei, ese, wq
➜mkdir merge-conflict
➜cd merge-conflict
➜touch conflict.txtcreate the conflict file
➜vim conflict.txtadd content → "welcome to ubuntu"
➜git init
➜git add .
➜git commit -m "my-first-ubuntu version1"
➜git log
➜cat conflict.txtverify: "welcome to ubuntu"
➜vim conflict.txtedit: "Linus I hate u" [ei, Esc, :wq]
➜git ls
➜git status
➜git switch -b devswitch to dev branch
➜git log --onelineshow log from main branch
➜git commit -m "Imy-1 version3 from main branch"→
➜git add .
➜git commit -m "Imy-1 main branch"→
➜git merge devmerge dev into main → CONFLICT!
➜cat conflict.txtsee conflict markers. Let's fix our own version.
➜vim conflict.txtresolve: i, Esc, :wq
➜git add .mark conflict resolved
➜git status
➜git commit -m "Imy-1 version3 from main branch"final merge commit
➜git log --onelineclean linear history
➜cat conflict.txtverify resolved content
➜vim conflict.txtconflict এর সমাধান করতে হয়
➜cat conflict.txtverify final
Conflict File Format
<<<<<<< HEAD your version of the line ======= their version of the line >>>>>>> dev
Fix: delete the markers, keep the correct code, then git add . and commit.
08 Undoing / Restoring Changes ▼
➜git statussee modified / deleted files
➜rm love-letter.txtaccidentally delete file
➜lsfile is gone
➜git statusgit shows it as deleted
➜git restore love-letter.txtrestore deleted file from last commit
➜cat love-letter.txtverify file is back
➜ls -ashow hidden files (for show hidden file)
➜cd .gitexplore .git internals
➜ls
➜cd ..go back (for back directory)
git is a version control system — it keeps every version, so you can always restore!
